The deliverable

The un-reviewed report,
on every pull request.

The one report no linter can produce: what a human hasn't reviewed. A unit goes STALE when its exact alpha-hash drifts, ORPHAN when a reviewed unit disappears. Both fail the gate; severity is advisory and never gates.

Graded trust · two axes

Turn the bar up
where it matters.

Billing might demand a principal human and an opus-class model on top. Dashboards might take an intern, or just CodeRabbit. One .vid/trust.toml holds two accept-list axes; only allowlisted reviewers count (off-allowlist reads unverified, fail-closed), and a path below its bar fails the gate — not just the report.

The CLI · seven commands

One gate,
every pull request.

The engine everything else renders. vid verify fails CI on orphaned and exact-hash-stale vouches, and on any scoped path below its required standing. The same core prints vid report, status, queue and emits the contract: vid json and vid sarif.

report status queue verify review json sarif

On GitHub · commitless

Vouch at the head SHA.
No commit. No re-run.

The primary flow is commitless: a reviewer runs /vouch and the GitHub App posts a required Check-run bound to the PR's head commit — identity from their verified GitHub session. No commit is added, the diff is untouched, CI is not re-triggered (the model Codecov and CodeQL use). The committed .vid/ ledger remains the offline, single-actor fallback. Merging never auto-vouches.

One engine · many languages

Ten languages, one config-driven engine.

tree-sitter is the universal frontend behind a single normalization profile — adding a language is a grammar plus a profile, not a new engine.

Rust · code Go · code Python · code JavaScript · code Java · code C# · code Kotlin · code Zig · code Swift · code Haskell · structural

Code-grade languages get full alpha-normalization (rename-, move-, reformat-invariant); structural-grade hashes the parse-tree shape. Rust lands first — vid reviews vid.

Works where you do.

The same one-gesture Vouch in every editor — each shells out to the vid binary and renders its coverage. None re-implements the engine; none edits your source.

An activity-bar coverage tree and status bar; a co-located CodeLens to Vouch any unit, or run the gate — all from the vid.json the binary emits.

The hosted platform · vici

Push your coverage. See it across every repo.

The free vid tool stays MIT. The hosted Vidi Vouch platform is the commercial layer, and it's live: vid push ships your vid.json over an authenticated, tenant-isolated API (POST /api/v1/ingest, per-tenant token) and the portal renders it.

SELF-SERVE · COVERAGE

Connect a repo, see coverage

Connect a repository and mint or revoke a CI token from the portal — no admin. Each repo shows its latest-snapshot coverage and per-repo snapshot history.

THE REPORT · THE MATRIX

The un-reviewed report & matrix

Every stale, orphaned, under- or un-reviewed unit, worst-first, with the exact CI gate verdict. The two-axis human×machine matrix plots each unit and draws the policy frontier (for repos whose CI emits the v3 axes contract; without it, an honest "awaiting v3" state).

ORG · TEAM / ENTERPRISE

Org rollups & paid plans

Cross-repo rollups, reviewer standing, a directory heat-tree, and an embeddable coverage badge. Team and Enterprise plans with tier entitlements bill through Stripe. Managed SSO/RBAC and private Rekor/Fulcio are on the enterprise roadmap.

⚠ Phase 1: human coverage is self-asserted and unverified — the platform renders point-in-time coverage, not cryptographic proof.

One gesture

The ⌃⌥V Vouch is the same action in every editor, never buried in a Problems panel.

Survives the refactor

Vouches attach to alpha-normalized units, not lines. Rename, move, extract: the vouch holds until meaning changes.

Never edits your code

Reviews go to the .vid/ ledger or a commitless Check-run — provenance beside the source, never injected into it.
